What i learned in this course of coursera for git and github

1. git stash Meaning: Temporarily saves your uncommitted changes so you can work on something else. Syntax: git stash git stash pop git stash list git stash apply Explanation: git stash → Save current changes git stash pop → Restore and remove from stash git stash list → Show all saved stashes 2. git rebase Meaning: Reapplies commits on top of another base branch (cleaner history than merge). Syntax: git rebase git rebase -i Explanation: Moves your branch commits onto another branch -i → interactive mode (edit, squash commits) 3. git log Meaning: Shows commit history. Syntax: git log git log --oneline git log --graph Explanation: --oneline → compact view --graph → shows branch structure 4. git config Meaning: Set Git configuration (username, email, editor). Syntax: git config --global user.name "Your Name" git config --global user.email "your@email.com" git config --list 5. git status Meaning: Shows current state of files. Syntax: git status 6. git add Meaning: Adds files to staging area. Syntax: git add git add . 7. git commit Meaning: Saves changes permanently. Syntax: git commit -m "message" 8. git push Meaning: Uploads local commits to GitHub. Syntax: git push origin 9. git pull Meaning: Fetch + merge changes from remote. Syntax: git pull origin 10. git clone Meaning: Copies a remote repository. Syntax: git clone 11. git branch Meaning: Manage branches. Syntax: git branch git branch git branch -d 12. git checkout (older) / git switch Meaning: Switch branches or restore files. Syntax: git checkout git switch 13. git merge Meaning: Combines branches. Syntax: git merge 14. git fetch Meaning: Downloads changes without merging. Syntax: git fetch 15. git reset Meaning: Undo changes. Syntax: git reset --soft HEAD~1 git reset --hard HEAD~1 16. git revert Meaning: Undo a commit safely (creates new commit). Syntax: git revert 17. git diff Meaning: Shows changes between files/commits. Syntax: git diff git diff 18. git remote Meaning: Manage remote repositories. Syntax: git remote -v git remote add origin 19. git tag Meaning: Marks versions/releases. Syntax: git tag v1.0 git push origin v1.0 20. git show Meaning: Shows details of a commit. Syntax: git show 21. Friendly Git Bash Commands (Shortcuts & Useful Tips) clear # Clear terminal pwd # Show current directory ls # List files cd # Change directory history # Show previous commands 22. Advanced Helpful Commands Squash commits git rebase -i HEAD~3 Amend last commit git commit --amend Stash specific files git stash push

GitHub Pages 1. Wiki Meaning: A space to create and maintain project documentation. Use: Write guides, setup instructions, FAQs Helps users and contributors understand the project Example use: Documentation for installation steps API usage guide 2. Project Meaning: A tool to organize and track work using boards (like Trello). Use: Manage tasks using columns like To Do, In Progress, Done Track progress of features or bugs Example: Sprint planning board for a team project 3. Insights Meaning: Analytics and statistics about your repository. Use: View contributions, commits, and activity Track project health and collaboration Example: See who contributed the most Monitor repo traffic 4. Actions Meaning: Automate workflows using CI/CD (Continuous Integration & Deployment). Use: Run tests automatically Deploy code when changes are pushed Example: Run tests on every push Auto-deploy to a server or cloud 5. Issues Meaning: A tool to track bugs, tasks, and feature requests. Use: Report bugs Discuss improvements Assign tasks to team members Example: “Fix login bug” issue “Add dark mode feature” request
